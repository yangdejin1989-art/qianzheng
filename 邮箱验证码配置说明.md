# 邮箱验证码功能配置说明

## 功能概述

已成功实现邮箱验证码功能，用于增强申请进度查询的安全性。用户需要通过邮箱验证码验证身份后才能查看申请详情。

## 主要特性

### 安全特性
- ✅ 邮箱格式验证
- ✅ 验证码有效期控制（10分钟）
- ✅ 发送频率限制（1分钟内只能发送一次）
- ✅ 验证尝试次数限制（最多5次）
- ✅ JWT token验证（30分钟有效期）
- ✅ 邮箱地址部分隐藏显示

### 用户体验
- ✅ 三步式验证流程
- ✅ 步骤指示器
- ✅ 倒计时重发功能
- ✅ 实时错误提示
- ✅ 加载状态显示

## 邮箱服务配置

### 方法一：使用Gmail（推荐）

1. **获取Gmail应用密码**
   - 登录Gmail账号
   - 进入"管理您的Google账户"
   - 选择"安全性" → "两步验证"
   - 启用两步验证后，选择"应用专用密码"
   - 生成应用专用密码

2. **设置环境变量**
   ```bash
   # Windows
   set EMAIL_USER=your-email@gmail.com
   set EMAIL_PASS=your-app-password

   # Linux/Mac
   export EMAIL_USER=your-email@gmail.com
   export EMAIL_PASS=your-app-password
   ```

### 方法二：使用QQ邮箱

1. **开启SMTP服务**
   - 登录QQ邮箱
   - 设置 → 账户 → POP3/IMAP/SMTP/Exchange/CardDAV/CalDAV服务
   - 开启SMTP服务，获取授权码

2. **修改服务器配置**
   ```javascript
   // 在 server/index.js 中修改邮件配置
   const transporter = nodemailer.createTransporter({
     service: 'qq', // 改为qq
     auth: {
       user: process.env.EMAIL_USER || 'your-email@qq.com',
       pass: process.env.EMAIL_PASS || 'your-authorization-code'
     }
   });
   ```

### 方法三：使用其他邮箱服务

```javascript
// 自定义SMTP配置
const transporter = nodemailer.createTransporter({
  host: 'smtp.your-provider.com',
  port: 587,
  secure: false,
  auth: {
    user: process.env.EMAIL_USER,
    pass: process.env.EMAIL_PASS
  }
});
```

## API接口说明

### 1. 发送验证码
```
POST /api/send-verification-code
Content-Type: application/json

{
  "email": "user@example.com",
  "phone": "13800138000",        // 可选
  "applyCode": "20241210KD0001"  // 可选
}
```

### 2. 验证验证码
```
POST /api/verify-email-code
Content-Type: application/json

{
  "email": "user@example.com",
  "code": "123456",
  "phone": "13800138000",        // 可选
  "applyCode": "20241210KD0001"  // 可选
}
```

### 3. 查询申请状态（带验证）
```
GET /api/status?token=jwt-token-here
```

## 数据库模型

### EmailVerification 集合
```javascript
{
  email: String,           // 邮箱地址
  code: String,           // 验证码
  phone: String,          // 关联手机号（可选）
  applyCode: String,      // 关联申请编码（可选）
  purpose: String,        // 用途（默认：status_query）
  attempts: Number,       // 尝试次数（最多5次）
  verified: Boolean,      // 是否已验证
  expiresAt: Date,       // 过期时间（10分钟）
  createdAt: Date        // 创建时间
}
```

## 测试步骤

1. **启动服务**
   ```bash

   cd kdai-broadband/server
   npm start

   # 启动前端服务
   cd kdai-broadband/client

   ```

2. **配置邮箱**

3. **测试流程**

   - 验证成功后查看申请详情

## 故障排除

### 常见问题

1. **邮件发送失败**
   - 检查邮箱凭据是否正确
   - 确认SMTP服务是否开启
   - 检查网络连接

2. **验证码收不到**
   - 检查垃圾邮件文件夹
   - 确认邮箱地址是否正确
   - 检查邮件服务商限制

3. **Token验证失败**
   - 检查JWT密钥配置
   - 确认token是否过期
   - 验证token格式是否正确
