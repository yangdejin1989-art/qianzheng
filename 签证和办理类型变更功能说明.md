# 签证和办理类型变更功能说明

## 📋 功能概述

管理员现在可以为客户更换签证类型和办理类型，系统将自动处理相关的材料清单和问题模板，确保数据的一致性。

---

## 🎯 两种变更场景

### 场景 1：更换签证类型（Package）

当客户申请错了签证类型，需要更换为其他签证类型时使用。

**示例**：
- 客户原本申请：美国签证
- 需要更换为：日本签证

**效果**：
- ✅ 清空原办理类型
- ✅ 清空所有材料清单
- ✅ 清空所有问题答案
- ✅ 需要重新选择新签证下的办理类型

---

### 场景 2：更换办理类型（Customer Type）

当客户的办理类型选错了，需要更换为同一签证下的其他办理类型时使用。

**示例**：
- 签证类型：美国签证（不变）
- 原办理类型：学生签证（F1）
- 更换为：商务签证（B1）

**效果**：
- ✅ 清空原办理类型的材料清单
- ✅ 清空原办理类型的问题答案
- ✅ 自动加载新办理类型的材料清单和问题模板

---

## 💡 操作指南

### 方法一：更换签证类型

#### 步骤 1：进入编辑模式

1. 打开订单详情页面
2. 点击右上角的"编辑"按钮

#### 步骤 2：选择新的签证类型

在"服务信息"区域：
```
┌────────────────────────────┐
│ 签证类型：[下拉选择框 ▼]  │
└────────────────────────────┘
```

从下拉框中选择新的签证类型。

#### 步骤 3：确认更换

点击"保存"按钮后，系统会弹出确认对话框：

```
⚠️ 注意：您正在更改签证类型

原签证类型：美国签证
新签证类型：日本签证

更改签证类型后，系统将：
• 清空当前的办理类型（学生签证）
• 清空已配置的材料清单
• 清空已配置的问题答案

您需要重新为客户选择办理类型。

确认要更改吗？
```

#### 步骤 4：重新选择办理类型

更换成功后，系统提示：
```
签证类型已更换！

请为客户重新选择办理类型，系统将自动关联新的材料清单和问题模板。
```

在页面下方的黄色警告框中选择新的办理类型。

---

### 方法二：更换办理类型

#### 步骤 1：点击"更换"按钮

在订单详情的"服务信息"区域：

```
┌─────────────────────────────────┐
│ 办理类型：✓ 学生签证  [更换]   │
└─────────────────────────────────┘
```

点击"更换"按钮。

#### 步骤 2：选择新的办理类型

弹出"更换办理类型"对话框：

```
┌──────────────────────────────────────┐
│  更换办理类型                        │
├──────────────────────────────────────┤
│                                      │
│  ⚠️ 注意：更换办理类型后，原材料清   │
│  单和问题答案将被清空！              │
│                                      │
│  当前办理类型：                      │
│  [学生签证]                          │
│                                      │
│  → 选择新的办理类型：                │
│  [请选择新的办理类型 ▼]             │
│                                      │
│  💡 选择后将自动加载新办理类型的      │
│     材料清单和问题模板               │
│                                      │
│         [取消]     [确认更换]        │
└──────────────────────────────────────┘
```

#### 步骤 3：确认更换

选择新的办理类型后，点击"确认更换"按钮。

系统会再次确认：

```
⚠️ 注意：您正在更换办理类型

原办理类型：学生签证
新办理类型：商务签证

更换办理类型后，系统将：
• 清空原办理类型的材料清单
• 清空原办理类型的问题答案
• 自动加载新办理类型的材料清单和问题模板

客户已上传的材料和已填写的答案将被清空！

确认要更换吗？
```

#### 步骤 4：查看更换结果

确认后，系统提示：

```
办理类型已更换！

原材料清单和问题答案已清空，新的材料清单和问题模板已自动关联。
```

页面自动刷新，显示新的：
- ✅ 办理类型
- ✅ 材料清单（按新办理类型生成）
- ✅ 问题模板（按新办理类型生成）

## 🔄 完整流程示例

### 示例 1：客户申请错了签证国家

**背景**：
- 客户：张三
- 原申请：美国签证 → 学生签证（F1）
- 已上传：护照、I-20表格等材料
- 已填写：10个问题的答案

**操作步骤**：

1. **发现问题**
   - 客户联系说申请错了，应该是日本签证

2. **更换签证类型**
   - 管理员点击"编辑"
   - 将"签证类型"从"美国签证"改为"日本签证"
   - 点击"保存"
   - 确认更换（知道会清空所有数据）

3. **重新配置**
   - 系统清空：办理类型、材料、问题
   - 管理员重新选择办理类型："留学签证"
   - 系统自动加载日本留学签证的材料清单和问题模板

4. **联系客户**
   - 通知客户重新准备材料
   - 通知客户重新填写问题

---

### 示例 2：客户选错了办理类型

**背景**：
- 客户：李四
- 签证类型：美国签证（正确）
- 原办理类型：学生签证（F1）（错误）
- 正确类型：商务签证（B1）
- 已上传：I-20表格、成绩单等学生签证材料
- 已填写：学校名称、专业等学生问题

**操作步骤**：

1. **确认问题**
   - 管理员联系客户确认
   - 客户确认应该是商务签证

2. **更换办理类型**
   - 在订单详情页面，办理类型右侧点击"更换"按钮
   - 在弹窗中选择"商务签证（B1）"
   - 点击"确认更换"
   - 确认清空原材料和问题

3. **自动处理**
   - 系统清空：学生签证的材料（I-20、成绩单等）
   - 系统清空：学生签证的问题（学校、专业等）
   - 系统自动加载：商务签证的材料清单（邀请函、公司证明等）
   - 系统自动加载：商务签证的问题模板（公司名称、访问目的等）

4. **通知客户**
   - 告知客户需要准备的新材料
   - 请客户重新填写商务签证相关问题

const handleSave = async () => {

### 后端实现

**位置**：`qianzheng/server/index.js`

#### 1. 处理签证类型更换

```javascript
// 处理签证类型更换
if (pkg !== undefined && packageChanged === true) {
  // 签证类型改变了，清空相关数据
  appItem.package = pkg;
  appItem.customerType = null; // 清空办理类型
  appItem.materials = []; // 清空材料清单
  appItem.questionsAnswers = []; // 清空问题答案
  console.log('✅ 签证类型已更换，已清空办理类型、材料清单和问题答案');
}
```

#### 2. 处理办理类型更换

```javascript
if (customerType !== undefined) {
  // 如果是更换办理类型，先清空原有的材料和问题
  if (customerTypeChanged === true) {
    appItem.materials = []; // 清空材料清单
    appItem.questionsAnswers = []; // 清空问题答案
    console.log('✅ 办理类型更换，已清空原材料清单和问题答案');
  }
  
  appItem.customerType = customerType;
  
  // 自动加载新的材料清单和问题模板
  if (customerType && customerType.typeId) {
    // 加载问题模板
    const questionTemplate = await QuestionTemplate.findOne({
      packageId: packageInfo._id,
      customerTypeId: customerType.typeId
    });
    
    if (questionTemplate) {
      appItem.questionsAnswers = questionTemplate.questions.map(...);
    }
    
    // 加载材料模板
    const materialTemplate = await MaterialTemplate.findOne({
      packageId: packageInfo._id
    });
    
    const customerTypeData = materialTemplate.customerTypes.find(
      ct => ct.typeId === customerType.typeId
    );
    
    if (customerTypeData) {
      // 为所有人员生成材料清单
      appItem.materials = generateMaterials(customerTypeData.materials);
    }
  }
}
```

## ⚠️ 重要注意事项

### 1. 数据清空不可恢复

- ❌ **材料清空后无法恢复**：客户已上传的材料将被永久删除
- ❌ **答案清空后无法恢复**：客户已填写的问题答案将被永久删除
- ⚠️ **操作前务必确认**：建议先联系客户确认再操作

### 2. 操作建议

#### 更换签证类型前：
1. ✅ 联系客户确认是否真的需要更换
2. ✅ 如有需要，先手动记录客户已上传的材料列表（在管理员备注中）
3. ✅ 记录客户已填写的问题答案（如有重要信息）
4. ✅ 通知客户需要重新准备材料

#### 更换办理类型前：
1. ✅ 确认客户的签证类型是正确的（不需要更换签证类型）
2. ✅ 联系客户确认新的办理类型
3. ✅ 提前告知客户需要重新准备哪些材料
4. ✅ 如有已上传的材料，先确认是否有可以保留的（手动记录）

### 3. 多人申请注意事项

- 更换办理类型会清空**所有人**的材料清单
- 包括：主申请人 + 所有同行人
- 更换后会自动为所有人重新生成新办理类型的材料清单

### 4. 模板配置要求

- ✅ 确保新的签证类型已配置办理类型和模板
- ✅ 确保新的办理类型已配置材料清单和问题模板
- ⚠️ 如未配置模板，更换后材料和问题列表将为空

## 📝 常见问题

### Q1: 更换签证类型后，能恢复原来的材料吗？

**答案**：
- ❌ 不能自动恢复
- 材料已被永久删除
- 建议操作前先在"管理员备注"中记录客户已上传的材料列表

---

### Q2: 更换办理类型和重新选择办理类型有什么区别？

**答案**：

| 场景 | 触发方式 | 清空数据 | 用途 |
|------|---------|---------|------|
| **重新选择** | 在警告框中首次选择 | ❌ 不清空 | 客户未选择办理类型时使用 |
| **更换** | 点击"更换"按钮 | ✅ 清空 | 客户选错办理类型时使用 |

---

### Q3: 更换后，处理记录会显示什么？

**答案**：

更换签证类型：
```
管理员更新：签证类型从"美国签证"更换为"日本签证"，
已清空原办理类型、材料清单和问题答案
```

更换办理类型：
```
管理员更新：办理类型从"学生签证"更换为"商务签证"，
已清空原材料和问题，已自动关联新的材料清单和问题模板
```

---

### Q4: 同行人的材料也会被清空吗？

**答案**：
- ✅ 是的
- 更换办理类型会清空**所有人**的材料
- 包括主申请人和所有同行人
- 更换后会为所有人重新生成新的材料清单

---

### Q5: 如果新的办理类型没有配置模板会怎样？

**答案**：
- 办理类型可以成功更换
- 但材料清单和问题列表将为空
- 需要先在"模板配置"中为该办理类型配置模板
- 然后再次更换办理类型以加载模板
