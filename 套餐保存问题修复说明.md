# 套餐保存问题修复说明

## 问题描述
在合并价格系统后，保存套餐时出现"操作失败"错误。

## 问题原因
**根本原因**：前后端数据不匹配

1. **前端**：已经移除了独立的 `price` 和 `originalPrice` 字段，不再发送这些数据
2. **后端**：仍然尝试处理这些字段，使用 `Number(price)` 和 `Number(originalPrice)`
3. **结果**：当这些字段为 `undefined` 时，`Number(undefined)` 返回 `NaN`，导致数据库保存失败

## 解决方案

### 1. 改进前端错误显示
**文件**：`qianzheng/client/src/PackageManager.js`

修改了错误处理，显示详细错误信息：
```javascript
catch (err) {
  console.error('保存失败:', err);
  console.error('错误响应:', err.response?.data);
  setError(err.response?.data?.message || '操作失败：' + (err.message || '未知错误'));
}
```

### 2. 修复后端数据处理
**文件**：`qianzheng/server/index.js`

#### 新增套餐接口（POST /api/packages）
修改前：
```javascript
price: Number(price),
originalPrice: Number(originalPrice),
```

修改后：
```javascript
price: price ? Number(price) : undefined,
originalPrice: originalPrice ? Number(originalPrice) : undefined,
```

#### 更新套餐接口（PUT /api/packages/:id）
同样的修改，让 `price` 和 `originalPrice` 变成可选字段。

## 修改说明

### 为什么这样修改？
1. **保持向后兼容**：旧套餐数据仍然有 `price` 和 `originalPrice` 字段
2. **支持新格式**：新套餐使用 `visaTypes` 数组，每个类型有自己的价格和币种
3. **防止 NaN**：只有在字段有值时才转换为数字，否则保持 `undefined`

### 数据结构对比

#### 旧格式（仍然支持）
```javascript
{
  name: "台湾签证",
  price: 12000,
  originalPrice: 15000,
  currency: "JPY"
}
```

#### 新格式（推荐）
```javascript
{
  name: "台湾签证",
  visaTypes: [
    {
      type: "单次",
      currency: "JPY",
      price: 12000,
      originalPrice: 15000
    },
    {
      type: "一年多次",
      currency: "CNY",
      price: 15000,
      originalPrice: 18000
    }
  ]
}
```
