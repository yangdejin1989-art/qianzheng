# 套餐保存失败根本原因修复

## 问题现象
1. 保存套餐时提示"服务器错误"
2. 填写的签证类型和价格无法保存
3. 前端显示不出币种信息

## 根本原因

### 🔴 核心问题：Mongoose Schema 定义错误

**文件**: `server/models/Package.js`

#### 错误的定义方式
```javascript
visaTypes: [{
  type: String,  // ❌ 问题所在！
  currency: String,
  price: Number,
  originalPrice: Number
}]
```

#### 为什么会出错？
在 Mongoose 中，当你在数组内直接定义对象，并且对象中有一个字段叫 `type` 时，Mongoose 会误认为：
- 这个 `type` 是在定义数组元素的数据类型
- 而不是一个字段名

这导致 Mongoose 试图将整个对象数组解析为字符串类型，造成类型转换错误：
```
Cast to [string] failed for value "[...]"
```

## 解决方案

### ✅ 正确的定义方式

创建一个单独的子 Schema，然后在主 Schema 中引用它：

```javascript
const mongoose = require('mongoose');

// 1. 先定义子 Schema
const visaTypeSchema = new mongoose.Schema({
  type: String,  // 现在可以正常使用了
  currency: String,
  price: Number,
  originalPrice: Number
}, { _id: false }); // 禁用 _id 字段

// 2. 在主 Schema 中使用
const packageSchema = new mongoose.Schema({
  name: String,
  speed: String,
  visaTypes: [visaTypeSchema], // ✅ 使用子 Schema
  // ... 其他字段
});

module.exports = mongoose.model('Package', packageSchema);
```

## 技术要点

### Mongoose Schema 最佳实践

1. **当数组元素是对象，且对象包含 `type` 字段时**：
   ```javascript
   ❌ 错误：
   items: [{
     type: String,
     value: Number
   }]
   
   ✅ 正确：
   const itemSchema = new Schema({ type: String, value: Number });
   items: [itemSchema]
   ```

2. **禁用子文档的 _id**：
   ```javascript
   new Schema({ ... }, { _id: false })
   ```
   这样可以避免 MongoDB 为每个子文档自动生成 ObjectId。

3. **使用独立的 Schema 的好处**：
   - 更清晰的代码结构
   - 避免字段名冲突
   - 可以添加子文档级别的方法和虚拟字段
   - 更容易维护和扩展
