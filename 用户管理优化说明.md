# 用户管理优化说明

## 📋 优化概述

根据实际使用场景，对用户管理和个人设置功能进行了优化，实现了管理员统一管理和员工个人设置分离的设计。

---

## ✨ 主要改进

### 1. 菜单权限优化

#### 管理员账户
- **不显示**"个人设置"菜单
- 在"用户管理"中可以编辑自己的信息
- 统一在一个地方管理所有用户（包括自己）

#### 员工账户
- **显示**"个人设置"菜单
- 可以修改自己的基本信息、联系方式和密码
- 无法访问"用户管理"（除非有权限）

---

## 🎯 功能详解

### 用户管理页面（管理员专属）

管理员可以在"用户管理"中：

#### 管理员账户（admin）
- ✅ 查看基本信息（用户名、显示名称）
- ✅ 编辑显示名称
- ✅ 编辑联系方式（手机、邮箱、微信、QQ）
- ✅ 修改密码
- ❌ 不显示权限设置（管理员拥有所有权限）
- ❌ 不能禁用或删除自己

#### 员工账户
- ✅ 查看所有信息
- ✅ 编辑显示名称
- ✅ 编辑联系方式（手机、邮箱、微信、QQ）
- ✅ 修改密码
- ✅ 配置权限
- ✅ 禁用/激活账户
- ✅ 删除账户

---

### 个人设置页面（员工专属）

员工可以在"个人设置"中：
- ✅ 修改显示名称
- ✅ 修改联系方式（手机、邮箱、微信、QQ）
- ✅ 修改密码

## 🔌 技术实现

### 前端修改

#### AdminPanel.js
1. 菜单项添加`staffOnly`标记
```javascript
{ key: 'account', label: '个人设置', icon: '●', staffOnly: true }
```

2. 菜单过滤逻辑
```javascript
const filteredMenuList = menuList.filter(menu => {
  // 管理员不显示staffOnly的菜单
  if (menu.staffOnly && isAdmin) return false;
  // 管理员显示所有非staffOnly的菜单
  if (isAdmin) return true;
  // 员工不显示adminOnly的菜单
  if (menu.adminOnly) return false;
  // staffOnly的菜单对员工显示
  if (menu.staffOnly) return true;
  // 没有权限要求的菜单都显示
  if (!menu.permission) return true;
  // 检查是否有对应权限
  return permissions[menu.permission] === true;
});
```

#### UserManagement.js
1. 添加联系方式字段到formData
```javascript
phone: '',
email: '',
wechat: '',
qq: ''
```

2. 编辑按钮对所有用户（包括管理员）显示
```javascript
<button className="btn btn-outline-primary" onClick={() => handleEdit(user)}>
  编辑
</button>
```

3. 表单中添加联系方式输入框
4. 权限设置只对非管理员显示
```javascript
{(!editingUser || editingUser.role !== 'admin') && (
  <div className="mb-3">
    <label className="form-label d-block">权限设置</label>
    ...
  </div>
)}
```

### 后端修改

#### server/index.js

##### 用户列表API
```javascript
app.get('/api/admin/users', checkPermission('userManagement'), async (req, res) => {
  // 返回数据中包含联系方式
  users: users.map(user => ({
    id: user._id,
    username: user.username,
    displayName: user.displayName,
    phone: user.phone,
    email: user.email,
    wechat: user.wechat,
    qq: user.qq,
    role: user.role,
    permissions: user.permissions,
    isActive: user.isActive,
    createdAt: user.createdAt
  }))
});
```

##### 用户更新API
```javascript
app.put('/api/admin/users/:id', checkPermission('userManagement'), async (req, res) => {
  // 接收联系方式字段
  const { displayName, phone, email, wechat, qq, permissions, isActive, password } = req.body;
  
  // 更新基本信息（所有用户）
  if (displayName !== undefined) user.displayName = displayName;
  if (phone !== undefined) user.phone = phone;
  if (email !== undefined) user.email = email;
  if (wechat !== undefined) user.wechat = wechat;
  if (qq !== undefined) user.qq = qq;
  
  // 管理员只能修改自己的基本信息和密码，不能修改权限和激活状态
  if (user.role !== 'admin') {
    if (permissions !== undefined) user.permissions = permissions;
    if (isActive !== undefined) user.isActive = isActive;
  }
  
  if (password) {
    user.password = await bcrypt.hash(password, 10);
  }
});
```

## 📊 使用场景

### 场景1：管理员修改自己的信息
1. 登录管理员账户
2. 点击左侧菜单"用户管理"
3. 在用户列表中找到自己（admin）
4. 点击"编辑"按钮
5. 修改显示名称、联系方式或密码
6. 点击"保存"

### 场景2：管理员管理员工
1. 登录管理员账户
2. 点击左侧菜单"用户管理"
3. 在用户列表中找到需要编辑的员工
4. 点击"编辑"按钮
5. 修改员工的信息、权限等
6. 点击"保存"

### 场景3：员工修改自己的信息
1. 登录员工账户
2. 点击左侧菜单"个人设置"
3. 在个人设置页面修改信息
4. 点击"保存基本信息"或"修改密码"

## 🚀 优势特点

### 统一管理
- ✅ 管理员在一个地方管理所有用户
- ✅ 无需在多个页面切换
- ✅ 更符合管理员的使用习惯

### 权限分离
- ✅ 员工只能看到和使用自己权限范围内的功能
- ✅ 员工可以管理自己的基本信息
- ✅ 管理员保持完全控制权

### 用户体验
- ✅ 界面简洁，功能明确
- ✅ 操作流程清晰
- ✅ 减少误操作

---

## ⚠️ 注意事项

1. **管理员账户**
   - 管理员不能禁用或删除自己
   - 管理员的权限不能修改（始终拥有所有权限）
   - 管理员可以修改自己的基本信息和密码

2. **员工账户**
   - 员工在"个人设置"中只能修改自己的信息
   - 员工的权限由管理员在"用户管理"中设置
   - 员工可以随时修改自己的密码

3. **密码修改**
   - 编辑时密码留空表示不修改
   - 密码长度至少6位
   - 修改后立即生效
