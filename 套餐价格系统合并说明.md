# 套餐价格系统合并说明

## 修改日期
2025年11月7日

## 问题描述
之前的套餐编辑表单存在两套价格输入系统：
1. 独立的"币种"、"价格"、"原价"输入框
2. "签证类型与价格"列表

这导致数据混乱，用户填写时不清楚应该使用哪个，且保存可能失败。

## 解决方案
将两套系统合并为一个统一的"签证类型与价格"系统，每个签证类型都可以设置自己的：
- 类型名称（如：单次、一年多次）
- 币种（人民币、日元、美元、欧元）
- 价格
- 原价（选填）

## 数据结构

### visaTypes 数组格式
```javascript
[
  {
    type: "单次",           // 签证类型名称
    currency: "CNY",       // 币种代码
    price: 600,            // 价格
    originalPrice: 800     // 原价（可选）
  },
  {
    type: "一年多次",
    currency: "JPY",
    price: 15000,
    originalPrice: 20000
  }
]
```

### 支持的币种
- CNY：人民币（¥）
- JPY：日元（¥）
- USD：美元（$）
- EUR：欧元（€）

## 使用说明

### 如何添加套餐
1. 点击"新增套餐"按钮
2. 填写套餐名称和预计完成时间
3. 在"签证类型与价格"部分：
   - 输入类型名称（如：单次）
   - 选择币种
   - 输入价格
   - （可选）输入原价
   - 点击"添加"按钮
4. 可以添加多个签证类型，每个类型可以有不同的币种
5. 填写简短描述、特色功能、详细说明等
6. 上传套餐图片
7. 点击"新增套餐"按钮保存

### 如何编辑签证类型
- 如需修改某个签证类型：先点击"删除"按钮删除它，然后重新添加
- 支持添加、删除多个签证类型
- 每个类型独立设置币种和价格

### 特色功能的显示
- 在"特色功能"部分添加的内容会显示在套餐卡片底部
- 主页卡片最多显示3个特色功能，如有更多会显示"+N"标记
- 点击"查看详情"可看到所有特色功能

## 后端兼容性
后端 Package 模型已经支持 visaTypes 数组，每个签证类型都有自己的 currency 字段，无需修改后端代码。

## 注意事项
1. 至少需要添加一个签证类型才能保存套餐
2. 每个签证类型的价格和币种都是独立的
3. 旧的套餐数据（使用单一 price/currency）仍然可以正常显示
4. 建议将所有套餐迁移到新的 visaTypes 格式以获得更好的灵活性

## 优势
1. ✅ 一个套餐可以支持多种签证类型
2. ✅ 每种类型可以使用不同的币种
3. ✅ 界面更清晰，避免混淆
4. ✅ 保存成功率提高
5. ✅ 客户查看时可以看到所有可选类型和对应价格
