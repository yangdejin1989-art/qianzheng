# 📱 手机端调试指南

## 方法1：浏览器开发者工具（推荐）⭐

### Chrome / Edge 浏览器

1. **打开开发者工具**
   - 按键盘 `F12`
   - 或右键点击页面 → 选择"检查"

2. **切换到设备模拟模式**
   - 点击左上角的**手机图标**（设备工具栏）
   - 或按快捷键 `Ctrl + Shift + M`

3. **选择设备**
   - 顶部下拉菜单选择设备：
     - iPhone 12 Pro
     - iPhone SE
     - iPad
     - Samsung Galaxy S20
     - Pixel 5
     - 或自定义尺寸

4. **查看效果**
   - 页面会自动调整为手机屏幕大小
   - 可以看到移动端的布局

### 常用手机尺寸参考

| 设备 | 宽度 | 高度 |
|------|------|------|
| iPhone SE | 375px | 667px |
| iPhone 12/13 | 390px | 844px |
| iPhone 12/13 Pro Max | 428px | 926px |
| Samsung Galaxy S20 | 360px | 800px |
| iPad | 768px | 1024px |

### 快捷操作
- **旋转屏幕**：点击旋转图标（横屏/竖屏切换）
- **缩放页面**：调整缩放比例（100%, 75%, 50%等）
- **触摸模拟**：鼠标点击会模拟手指触摸

## 方法2：直接用真实手机访问

### 步骤

1. **确保手机和电脑在同一WiFi网络**

2. **找到电脑IP地址**
   - 打开命令提示符（Win + R，输入 cmd）
   - 输入命令：`ipconfig`
   - 找到 "IPv4 地址"，例如：`192.168.1.100`

3. **在手机浏览器访问**
   - 打开手机浏览器
   - 输入：`http://你的IP地址:3000`
   - 例如：`http://192.168.1.100:3000`

4. **查看实际效果**
   - 这是最真实的效果
   - 可以测试触摸、滑动等交互

### 注意事项
- 确保防火墙允许端口 3000
- 前端开发服务器必须正在运行
- 有些路由器可能需要特殊配置

## 当前项目的响应式断点

查看 `client/src/App.js`，项目使用 Bootstrap 的响应式系统：

```javascript
// 移动端判断
const [isMobile, setIsMobile] = useState(false);

useEffect(() => {
  const checkMobile = () => {
    setIsMobile(window.innerWidth <= 768);
  };
  checkMobile();
  window.addEventListener('resize', checkMobile);
}, []);
```

### Bootstrap 断点参考
- **xs**: < 576px（超小屏幕，手机竖屏）
- **sm**: ≥ 576px（小屏幕，手机横屏）
- **md**: ≥ 768px（中等屏幕，平板竖屏）
- **lg**: ≥ 992px（大屏幕，平板横屏/小笔记本）
- **xl**: ≥ 1200px（超大屏幕，桌面）

## 常见移动端问题检查清单

### 布局问题 ✓
- [ ] 页面内容是否超出屏幕宽度
- [ ] 文字大小是否适合手机阅读（最小 14px）
- [ ] 按钮是否足够大（最小 44x44px）
- [ ] 图片是否响应式缩放
- [ ] 表格是否可以横向滚动

### 导航问题 ✓
- [ ] 导航菜单是否变为汉堡菜单
- [ ] 导航栏是否固定在顶部
- [ ] 导航项是否容易点击

### 表单问题 ✓
- [ ] 输入框大小是否合适
- [ ] 键盘弹出时是否遮挡内容
- [ ] 下拉选择器是否易用

### 卡片布局 ✓
- [ ] 套餐卡片是否单列显示
- [ ] 卡片间距是否合适
- [ ] 图片比例是否正确

### 性能问题 ✓
- [ ] 图片是否过大（影响加载速度）
- [ ] 是否有不必要的动画
- [ ] 字体加载是否影响渲染

## 常见问题

**Q: 为什么我的页面在手机上和电脑上显示不一样？**  
A: 这是正常的！项目使用了响应式设计，会根据屏幕大小自动调整布局。

**Q: 如何知道某个元素在手机上的大小？**  
A: 在开发者工具中右键点击元素 → 检查 → 查看 Computed 标签的尺寸。

**Q: 模拟器显示和真实手机有差异怎么办？**  
A: 浏览器模拟器 90% 准确，但最终应在真实手机上测试确认。

**Q: 如何快速测试多个手机尺寸？**  
A: 使用自定义尺寸功能，或使用 Responsively App 同时预览多个设备。
