# 问题模板自动加载功能说明

## 🎯 功能概述

**问题：** 之前，如果客户在前端没有填写问题答案，后台订单详情中也不会显示问题模板，导致客服无法知道需要问哪些问题。

**解决方案：** 现在，只要客户选择了"客户类型"（办理类型），系统会自动加载该类型对应的问题模板到订单中，即使客户没有填写答案，后台也能看到所有问题。

## ✨ 新功能特性

### 1. 自动加载问题模板

当客户提交申请时：
- ✅ 如果选择了"客户类型"（如：学生签证、商务签证等）
- ✅ 系统自动从数据库加载该类型的问题模板
- ✅ 将模板中的所有问题保存到订单中
- ✅ 客户填写的答案会自动合并到对应问题
- ✅ 未填写的问题答案为空，但问题本身会保留

### 2. 后台显示优化

后台订单详情页面现在会：
- ✅ 始终显示问题答案区域（有客户类型时）
- ✅ 显示填写进度：已填写 X/总数 个问题
- ⚠️ 如果全部未填写，显示"客户未填写，请客服补充"警告
- ✓ 如果有部分填写，显示填写进度
- 📝 每个问题都会显示，未填写的标记为"未填写"

### 3. 数据结构

每个问题包含以下字段：

```javascript
{
  questionId: "q_001",              // 问题ID
  questionText: "您的出生日期是？",  // 问题文本
  questionType: "personal",          // 问题类型（personal/common）
  required: false,                   // 是否必填
  helpText: "请填写YYYY-MM-DD格式",  // 帮助文本
  answer: "",                        // 答案（空表示未填写）
  answeredBy: "user",                // 谁填写的（user/admin）
  answeredAt: "2025-10-29T..."       // 填写时间
}
```

## 🔧 技术实现

### 后端修改（server/index.js）

在申请提交接口中添加了问题模板加载逻辑：

```javascript
// 如果有客户类型，自动加载问题模板
if (customerType) {
  const QuestionTemplate = require('./models/QuestionTemplate');
  const questionTemplate = await QuestionTemplate.findOne({
    packageId: pkg,
    customerTypeId: customerType.typeId
  });
  
  if (questionTemplate && questionTemplate.questions) {
    // 创建完整的问题列表
    const fullQuestions = questionTemplate.questions.map(templateQuestion => {
      // 查找用户是否填写了这个问题的答案
      const userAnswer = questionsAnswers.find(qa => qa.questionId === templateQuestion.questionId);
      
      return {
        questionId: templateQuestion.questionId,
        questionText: templateQuestion.questionText,
        questionType: templateQuestion.questionType,
        required: templateQuestion.required,
        helpText: templateQuestion.helpText,
        answer: userAnswer ? userAnswer.answer : '', // 用户未填写则为空
        answeredBy: userAnswer ? 'user' : '',
        answeredAt: userAnswer ? new Date() : null
      };
    });
    
    applicationData.questionsAnswers = fullQuestions;
  }
}
```

5. 补充所有问题的答案
6. 保存

## 🐛 故障排除

### 问题：后台没有显示问题模板

**原因：**
1. 该签证类型+客户类型没有配置问题模板
2. 客户没有选择客户类型
3. 数据库连接问题

**解决：**
1. 检查是否已在后台配置问题模板
2. 确认客户申请时选择了客户类型
3. 检查后端日志是否有错误

### 问题：问题显示不完整

**原因：**
- 问题模板配置不完整

**解决：**
- 后台管理 → 问题模板管理 → 检查并补充问题

## 📝 注意事项

1. **必须选择客户类型**
   - 只有选择了客户类型，才会自动加载问题模板

2. **问题模板需要预先配置**
   - 在后台管理中配置好问题模板
   - 配置后立即生效

3. **客服可以随时编辑**
   - 无论客户是否填写
   - 客服都可以补充或修改答案
