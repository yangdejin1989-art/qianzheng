## 🔍 需要检查的问题

### 1. 浏览器控制台错误

请按以下步骤检查：

1. 打开浏览器（Chrome/Edge等）
2. 按 `F12` 打开开发者工具
3. 点击 "Console（控制台）" 标签
4. 尝试提交申请
5. 查看是否有**红色错误信息**

**常见错误及解决方案：**

#### 错误 A: "Network Error" 或 "ERR_CONNECTION_REFUSED"
- **原因**：前端无法连接到后端
- **解决**：
  ```bash
  # 1. 检查后端是否运行
  netstat -ano | findstr :5000
  
  # 2. 如果没有运行，启动后端
  cd qianzheng
  一键启动.bat
  ```

#### 错误 B: "Failed to fetch"
- **原因**：浏览器CORS策略阻止
- **解决**：检查你访问的URL是 `http://localhost:3000` 而不是 `http://127.0.0.1:3000`

#### 错误 C: "Request failed with status code 400"
- **原因**：表单数据验证失败
- **查看**：在控制台中查看具体的错误信息

### 2. 表单填写完整性检查

请确保以下必填项都已填写：

- [ ] **姓名** - 必填
- [ ] **手机号码** - 必填（8-15位数字，例如：09012345678）
- [ ] **联系地址** - 必填
- [ ] **签证类型** - 必填
- [ ] **微信号或LINE号** - 至少填一个
- [ ] **验证码** - 必填（4位，需要与图片验证码一致）

**验证码注意事项：**
- 验证码区分大小写
- 如果看不清可以点击刷新按钮
- 必须完全匹配才能提交

### 3. 网络请求详细信息

在浏览器开发者工具中：

1. 点击 "Network（网络）" 标签
2. 勾选 "Preserve log（保留日志）"
3. 尝试提交申请
4. 查找名为 "apply" 的请求
5. 点击查看详细信息：
   - **Status（状态码）**：应该是 200（成功）或 400/500（错误）
   - **Response（响应）**：查看服务器返回的错误信息
   - **Preview（预览）**：查看格式化的响应数据

## 🎯 最可能的原因

根据经验，最常见的问题是：

1. **验证码输入错误** - 50%的概率

   - 解决：换个浏览器试试（Chrome、Edge、Firefox）
