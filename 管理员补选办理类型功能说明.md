# 管理员补选办理类型功能说明

## 📋 功能概述

当客户在提交申请时没有选择办理类型时，管理员现在可以直接在订单详情页面为客户补选办理类型，系统将自动关联相应的材料清单和问题模板。

---

## 🎯 使用场景

### 场景说明

客户在申请签证时，可能因为以下原因未选择办理类型：
- ✅ 客户不确定自己的签证类型
- ✅ 客户想先提交申请，后续再补充详细信息
- ✅ 客户跳过了办理类型选择步骤

在这种情况下，订单详情页面会显示：
```
⚠️ 客户未选择办理类型
```

---

## 💡 操作指南

### 步骤 1：查看订单详情

1. 登录管理后台
2. 点击"申请管理"
3. 找到显示"⚠️ 客户未选择"的订单
4. 点击查看订单详情

### 步骤 2：选择办理类型

在订单详情页面，您会看到一个黄色警告框：

```
⚠️ 客户未选择办理类型

客户在申请时没有选择具体的办理类型（如：学生签证、商务签证等），
因此系统无法显示材料清单。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
👉 快速选择办理类型：

[下拉选择框：-- 请选择办理类型 --] [确认选择]

💡 选择后将自动关联该类型的材料清单和问题模板
```

### 步骤 3：确认选择

1. 从下拉框中选择合适的办理类型（如：学生签证、商务签证、旅游签证等）
2. 点击"确认选择"按钮
3. 系统会弹出确认对话框：
   ```
   确认为客户选择办理类型"学生签证"吗？
   
   选择后将自动关联相应的材料清单和问题模板。
   ```
4. 点击"确定"

### 步骤 4：系统自动处理

确认后，系统会自动执行以下操作：

#### ✅ 1. 更新办理类型
- 订单的办理类型字段更新为选择的类型
- 在"基本信息"中显示办理类型标签变为绿色 ✓

#### ✅ 2. 自动关联材料清单
- 根据办理类型加载对应的材料清单模板
- 为主申请人和所有同行人生成完整的材料清单
- 材料状态默认为"未提交"
- 保留客户已上传的材料（如有）

#### ✅ 3. 自动关联问题模板
- 根据办理类型加载对应的问题模板
- 生成完整的问题列表
- 保留客户已填写的答案（如有）
- 未填写的问题显示为空，管理员可后续补充

#### ✅ 4. 记录操作日志
- 在订单的处理记录中添加日志：
  ```
  管理员更新：办理类型从"未选择"改为"学生签证"，
  已自动关联材料清单和问题模板
  ```

## 🔧 技术实现

### 前端功能

**位置**：`qianzheng/client/src/ApplicationDetail.js`

#### 1. 获取可用的办理类型列表
```javascript
// 根据签证类型自动加载办理类型列表
useEffect(() => {
  const fetchCustomerTypes = async () => {
    if (!application || !application.package) return;
    
    // 根据签证类型获取材料模板，其中包含办理类型列表
    const pkg = packages.find(p => p.name === application.package);
    if (pkg) {
      const response = await axios.get(`/api/material-templates/package/${pkg._id}`);
      setCustomerTypes(response.data.customerTypes);
    }
  };
  
  fetchCustomerTypes();
}, [application, packages]);
```

#### 2. 处理办理类型选择
```javascript
const handleCustomerTypeChange = async (typeId) => {
  // 获取选择的类型
  const selectedType = customerTypes.find(t => t.typeId === typeId);
  
  // 确认对话框
  if (confirm(`确认为客户选择办理类型"${selectedType.typeName}"吗？`)) {
    // 调用API更新
    await axios.put(`/api/applications/${application.id}`, {
      customerType: {
        typeId: selectedType.typeId,
        typeName: selectedType.typeName
      }
    });
    
    // 刷新页面以显示新的材料清单和问题模板
    await fetchApplication();
  }
};
```

### 后端功能

**位置**：`qianzheng/server/index.js` - `PUT /api/applications/:id`

#### 1. 接收customerType参数
```javascript
const { customerType } = req.body;

if (customerType !== undefined) {
  appItem.customerType = customerType;
  
  // 自动加载模板
  if (customerType && customerType.typeId) {
    // ... 加载逻辑
  }
}
```

#### 2. 自动加载问题模板
```javascript
const QuestionTemplate = require('./models/QuestionTemplate');
const questionTemplate = await QuestionTemplate.findOne({
  packageId: packageInfo._id,
  customerTypeId: customerType.typeId
});

if (questionTemplate && questionTemplate.questions) {
  // 保留已有答案，加载新问题
  const fullQuestions = questionTemplate.questions.map(templateQuestion => {
    const existingAnswer = appItem.questionsAnswers.find(
      qa => qa.questionId === templateQuestion.questionId
    );
    
    return {
      questionId: templateQuestion.questionId,
      questionText: templateQuestion.questionText,
      answer: existingAnswer ? existingAnswer.answer : '',
      // ... 其他字段
    };
  });
  
  appItem.questionsAnswers = fullQuestions;
}
```

#### 3. 自动加载材料清单
```javascript
const MaterialTemplate = require('./models/MaterialTemplate');
const materialTemplate = await MaterialTemplate.findOne({ 
  packageId: packageInfo._id 
});

const customerTypeData = materialTemplate.customerTypes.find(
  ct => ct.typeId === customerType.typeId
);

if (customerTypeData && customerTypeData.materials) {
  // 为所有人员生成材料清单
  const allPersons = [
    { id: 'main', name: appItem.name },
    ...appItem.companions.map((name, idx) => ({ 
      id: `companion_${idx}`, 
      name 
    }))
  ];
  
  const newMaterials = [];
  customerTypeData.materials.forEach(materialTemplate => {
    allPersons.forEach(person => {
      // 保留已有材料，添加新材料
      const existingMaterial = appItem.materials.find(m => 
        m.materialId === materialTemplate.materialId && 
        m.personId === person.id
      );
      
      if (existingMaterial) {
        newMaterials.push(existingMaterial);
      } else {
        newMaterials.push({
          materialId: materialTemplate.materialId,
          materialName: materialTemplate.name,
          personId: person.id,
          personName: person.name,
          status: '未提交',
          images: []
        });
      }
    });
  });
  
  appItem.materials = newMaterials;
}
```

## ⚠️ 注意事项

### 1. 数据保留
- ✅ **已上传的材料不会丢失**：选择办理类型后，客户已上传的材料会被保留
- ✅ **已填写的答案不会丢失**：客户已填写的问题答案会被保留
- ✅ **只添加新的材料和问题**：系统只会添加模板中新的材料项和问题项

### 2. 重复选择
- 如果需要更改办理类型，可以联系技术人员
- 当前版本不支持直接在界面上重新选择办理类型
- 建议：首次选择时务必确认准确

### 3. 模板配置
- 确保已在"模板配置"中为该签证类型配置了办理类型和对应的材料清单
- 确保已配置了问题模板
- 如果没有配置模板，选择办理类型后不会自动生成材料清单和问题列表

### 4. 同行人材料
- 系统会自动为主申请人和所有同行人生成材料清单
- 每个人的材料项是独立的
- 在"申请材料"页签可以切换不同人员查看和上传材料

## 🔍 常见问题

### Q1: 如果没有看到"快速选择办理类型"下拉框怎么办？

**可能原因：**
- 该签证类型还没有配置办理类型和材料模板
- 需要先在"模板配置"中为该签证类型添加办理类型

**解决方法：**
1. 进入"模板配置" → "材料模板管理"
2. 找到对应的签证类型
3. 添加办理类型和材料清单
4. 刷新订单详情页面

---

### Q2: 选择办理类型后，材料清单为什么是空的？

**可能原因：**
- 该办理类型下没有配置材料清单

**解决方法：**
1. 进入"模板配置" → "材料模板管理"
2. 找到对应的签证类型和办理类型
3. 为该办理类型添加材料项
4. 重新选择办理类型（需联系技术人员）

---

### Q3: 可以更改已选择的办理类型吗？

**答案：**
- 当前版本不支持在界面上直接更改
- 如需更改，请联系技术人员
- 建议：首次选择时务必确认准确

---

### Q4: 客户已上传的材料会丢失吗？

**答案：**
- ❌ 不会丢失！
- 系统会保留客户已上传的所有材料
- 只会添加新的材料项，不会删除已有材料

---

### Q5: 如何知道办理类型已成功更新？

**确认方法：**
1. 订单基本信息中，办理类型从"⚠️ 客户未选择"变为"✓ [类型名称]"（绿色）
2. 申请材料区域显示完整的材料清单
3. 问题答案区域显示完整的问题列表
4. 系统提示"办理类型已更新！材料清单和问题模板已自动关联。"
5. 处理记录中有操作日志
