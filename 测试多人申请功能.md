## 🧪 测试场景

### 测试场景1：查看多人申请详情（管理员）

#### 步骤：
1. 启动系统
   ```bash
   cd qianzheng
   一键启动.bat
   ```

2. 登录管理后台
   - 访问：`http://localhost:3000/admin`
   - 登录账号

3. 找到一个有同行人的申请记录
   - 在申请列表中，查找"含X位同行人"的徽章

4. 点击查看详情

#### 预期结果：

**人员标签栏**
- 看到人员标签：`[张三（主申请人）] [李四] [王五]`
- 主申请人标签默认选中（蓝色背景）

**共同信息区域**（如果有）
- 蓝色背景区域
- 标题："共同信息（所有人适用）"
- 显示旅行目的、居住地址等

**申请材料区域**
- 标题显示："申请材料 - 张三"
- 显示张三的护照、照片等材料

**个人问题答案区域**
- 标题显示："个人问题答案 - 张三"
- 显示张三的个人信息

#### 切换人员测试：
1. 点击"李四"标签
   - 标签变为蓝色选中状态
   - 材料区域标题变为："申请材料 - 李四"
   - 问答区域标题变为："个人问题答案 - 李四"
   - 显示李四的材料和问答

2. 点击"王五"标签
   - 同样切换显示王五的内容

#### 继承标识测试：
如果李四的某个答案是从张三继承的：
- 答案背景显示为黄色
- 显示蓝色徽章："继承自主申请人"

## 🐛 常见问题排查

### 问题1：点击人员标签没有反应
**检查：**
- 浏览器控制台是否有错误
- `activePersonIndex` 状态是否正确更新
- 数据中是否有 `materialsByPerson` 和 `questionsByPerson`

**解决：**
```javascript
// 在 ApplicationDetail.js 中添加调试日志
console.log('activePersonIndex:', activePersonIndex);
console.log('materialsByPerson:', application.materialsByPerson);
```

---

### 问题2：材料显示为空
**检查：**
- 数据是否已迁移
- `materialsByPerson` 数组是否正确
- `personId` 是否匹配

**解决：**
```javascript
// 检查数据
const currentPersonId = activePersonIndex === 0 ? 'main' : `comp${activePersonIndex}`;
console.log('currentPersonId:', currentPersonId);
const personMaterials = application.materialsByPerson?.find(mp => mp.personId === currentPersonId);
console.log('personMaterials:', personMaterials);
```

---

### 问题3：旧申请显示异常
**检查：**
- 是否运行了迁移脚本
- 兼容性代码是否正确

**解决：**
```javascript
// 使用兼容性降级
const materialsToShow = personMaterials?.materials || application.materials || [];
```
