# 权限管理系统使用说明

## 📋 功能概述

已完成完整的权限管理系统，包括：
- 管理员/员工角色区分
- 细粒度权限控制
- 员工账户管理
- 前后端权限验证

## 👥 角色说明

### 管理员（admin）
- 拥有所有功能的访问权限
- 可以创建、编辑、删除员工账户
- 可以为员工分配权限
- 不能被编辑或删除

### 员工（staff）
- 只能访问被授权的功能
- 默认只有订单管理权限
- 不能访问用户管理功能
- 可以被管理员编辑和删除

## 🔐 权限列表

| 权限名称 | 权限代码 | 说明 |
|---------|---------|------|
| 订单管理 | orderManagement | 查看和处理客户订单 |
| 套餐管理 | packageManagement | 管理签证类型和套餐 |
| 模板管理 | templateManagement | 管理材料和问题模板 |
| 黑名单管理 | blacklistManagement | 管理黑名单 |
| 常见问题管理 | faqManagement | 管理FAQ |
| 公告管理 | noticeManagement | 管理公告 |
| 用户管理 | userManagement | 管理员工账户（仅管理员） |

## 🚀 使用指南

### 1. 首次登录

系统中需要至少有一个管理员账户。如果还没有，需要通过数据库或其他方式创建：

```javascript
// 在MongoDB中直接创建管理员账户
{
  username: "admin",
  password: "$2b$10$...", // 加密后的密码
  role: "admin",
  displayName: "系统管理员",
  isActive: true
}
```

或使用Node.js脚本创建：

```javascript
const bcrypt = require('bcrypt');
const Admin = require('./models/Admin');

async function createAdmin() {
  const hashedPassword = await bcrypt.hash('你的密码', 10);
  const admin = new Admin({
    username: 'admin',
    password: hashedPassword,
    role: 'admin',
    displayName: '系统管理员',
    isActive: true
  });
  await admin.save();
  console.log('管理员创建成功！');
}

createAdmin();
```

### 2. 创建员工账户

管理员登录后：

1. 点击左侧菜单**"用户管理"**
2. 点击右上角**"创建员工账户"**按钮
3. 填写信息：
   - 用户名：登录用的账号（不能修改）
   - 显示名称：显示在界面上的名字
   - 密码：初始密码
   - 权限设置：勾选需要授予的权限
4. 点击**"创建"**

**建议：**
- 员工账户默认只给"订单管理"权限
- 根据岗位需要逐步授权
- 初始密码建议简单，让员工登录后自己修改

### 3. 编辑员工权限

1. 在用户列表中找到员工
2. 点击**"编辑"**按钮
3. 可以修改：
   - 显示名称
   - 密码（留空则不修改）
   - 权限设置
4. 点击**"保存"**

### 4. 禁用/激活账户

- 点击员工旁边的**"禁用"**按钮，该员工将无法登录
- 禁用后可以点击**"激活"**重新启用
- 禁用不会删除账户数据

### 5. 删除员工账户

- 点击**"删除"**按钮
- 确认后会永久删除该账户
- 管理员账户不能被删除

## 💡 菜单显示规则

员工登录后，只会看到自己有权限的菜单项：

| 菜单项 | 需要的权限 |
|--------|-----------|
| 申请管理 | orderManagement |
| 签证类型管理 | packageManagement |
| 材料与问题管理 | templateManagement |
| 产品简介管理 | packageManagement |
| 网站设置 | packageManagement |
| 黑名单管理 | blacklistManagement |
| 日志管理 | orderManagement |
| 数据统计 | orderManagement |
| 用户管理 | userManagement（仅管理员） |

**示例：**
- 只有订单管理权限的员工：只能看到"申请管理"、"日志管理"、"数据统计"
- 完全权限的员工：可以看到除"用户管理"外的所有菜单

## 🔒 安全特性

### 1. 前端权限控制
- 登录时获取用户权限信息
- 根据权限过滤菜单显示
- 无权限的页面不会显示

### 2. 后端权限验证
- 所有API都有token验证
- 敏感操作需要权限检查
- 管理员操作有特殊验证

### 3. 账户安全
- 密码使用bcrypt加密存储
- JWT token有效期2小时
- 禁用的账户无法登录

## 📝 常见场景

### 场景1：新员工入职

1. 管理员创建员工账户
2. 只给"订单管理"权限
3. 告知员工初始账号密码
4. 员工登录后处理订单

### 场景2：员工晋升

1. 管理员编辑员工信息
2. 增加更多权限（如套餐管理、模板管理）
3. 保存后员工刷新页面即可看到新菜单

### 场景3：员工离职

1. 先点击"禁用"停止其访问
2. 确认无问题后点击"删除"
3. 账户彻底移除

### 场景4：忘记密码

1. 管理员在编辑页面
2. 输入新密码
3. 保存即可，员工用新密码登录

## ⚠️ 注意事项

1. **管理员账户保护**
   - 管理员账户不能被编辑、禁用或删除
   - 请妥善保管管理员密码
   - 建议至少保留一个管理员账户

2. **权限授予原则**
   - 最小权限原则：只给必要的权限
   - 定期审查员工权限
   - 员工离职及时禁用账户

3. **密码安全**
   - 创建账户时使用强密码
   - 建议员工定期修改密码
   - 不要共享账户

4. **数据备份**
   - 定期备份数据库
   - 删除员工前确认其处理的订单状态
   - 重要操作前做好记录

## 🔧 技术细节

### 数据库结构

```javascript
Admin {
  username: String,        // 用户名
  password: String,        // 加密密码
  role: String,           // admin/staff
  displayName: String,    // 显示名称
  permissions: {
    orderManagement: Boolean,
    packageManagement: Boolean,
    templateManagement: Boolean,
    blacklistManagement: Boolean,
    faqManagement: Boolean,
    noticeManagement: Boolean,
    userManagement: Boolean
  },
  isActive: Boolean,      // 是否激活
  createdAt: Date,
  updatedAt: Date
}
```

### API接口

- `GET /api/admin/users` - 获取用户列表
- `POST /api/admin/users` - 创建员工
- `PUT /api/admin/users/:id` - 更新员工
- `DELETE /api/admin/users/:id` - 删除员工
- `POST /api/admin/login` - 登录（返回权限信息）

### 权限验证中间件

```javascript
checkPermission('orderManagement') // 订单相关API
checkPermission('userManagement')  // 用户管理API
// ... 其他权限验证
```
