# 办理类型更换 - 材料智能合并功能说明

## 📋 功能概述

当管理员更换客户的办理类型时，系统会智能合并材料清单：
- ✅ **保留相同材料的图片**：如护照、照片等两种办理类型都需要的通用材料
- ❌ **删除特有材料**：仅属于原办理类型的材料将被移除
- ✅ **添加新材料**：新办理类型特有的材料将被添加
- ❌ **清空问题答案**：所有问题答案将被清空

---

## 🎯 智能合并原理

### 材料匹配规则

系统通过 `materialId` 来判断两种办理类型的材料是否相同：

```
学生签证材料：               商务签证材料：
├─ passport (护照首页)       ├─ passport (护照首页)      ← 相同 ✅
├─ photo (2寸白底照片)       ├─ photo (2寸白底照片)      ← 相同 ✅
├─ i20 (I-20表格)           ├─ invitation (邀请函)      ← 不同
├─ transcript (成绩单)       ├─ company (公司证明)       ← 不同
└─ financial (资金证明)      └─ itinerary (行程单)       ← 不同
```

**更换结果：**
- ✅ 保留：护照首页、照片（图片都保留）
- ❌ 删除：I-20表格、成绩单、资金证明
- ✅ 添加：邀请函、公司证明、行程单

- ❌ I-20和资金证明的图片被删除（4张图片）

## 🔧 技术实现细节

### 后端智能合并逻辑

**位置：**`qianzheng/server/index.js`

```javascript
// 1. 获取现有材料
const existingMaterials = appItem.materials || [];
const newMaterials = [];
let preservedCount = 0; // 保留的材料数量

// 2. 遍历新办理类型的材料模板
customerTypeData.materials.forEach(materialTemplate => {
  allPersons.forEach(person => {
    
    // 3. 查找是否已有这个材料（根据materialId和personId匹配）
    const existingMaterial = existingMaterials.find(m => 
      m.materialId === materialTemplate.materialId && 
      m.personId === person.id
    );
    
    if (existingMaterial) {
      // 4. 如果找到相同材料，保留它（包括图片）
      newMaterials.push({
        ...existingMaterial, // 保留所有字段（包括images）
        materialName: materialTemplate.name, // 更新材料名称
        templateRequired: materialTemplate.required,
        needsImage: materialTemplate.needsImage,
        allowMultiple: materialTemplate.allowMultiple
      });
      
      // 统计保留的材料数量
      if (existingMaterial.images && existingMaterial.images.length > 0) {
        preservedCount++;
      }
    } else {
      // 5. 如果是新材料，创建新记录
      newMaterials.push({
        materialId: materialTemplate.materialId,
        materialName: materialTemplate.name,
        personId: person.id,
        personName: person.name,
        status: '未提交',
        images: [],
        // ... 其他字段
      });
    }
  });
});

// 6. 替换材料清单
appItem.materials = newMaterials;

// 7. 记录日志
console.log(`✅ 已自动加载材料清单: ${newMaterials.length} 项，保留了 ${preservedCount} 项已上传的材料`);
```

### 匹配条件

材料被视为"相同"的条件：

1. **materialId 相同**：材料ID必须完全一致
2. **personId 相同**：必须是同一个人的材料

```javascript
const isMatch = 
  existingMaterial.materialId === newMaterial.materialId && 
  existingMaterial.personId === newMaterial.personId;
```

### 保留的字段

当材料匹配时，以下字段会被保留：

| 字段 | 说明 | 是否保留 |
|------|------|---------|
| `images` | 已上传的图片列表 | ✅ 保留 |
| `status` | 材料状态（已提交/未提交） | ✅ 保留 |
| `submittedBy` | 提交者 | ✅ 保留 |
| `submittedAt` | 提交时间 | ✅ 保留 |
| `note` | 备注 | ✅ 保留 |
| `materialName` | 材料名称 | ❌ 更新为新名称 |
| `templateRequired` | 是否必需 | ❌ 更新为新配置 |
| `needsImage` | 是否需要图片 | ❌ 更新为新配置 |

## ⚠️ 重要注意事项

### 1. 材料ID必须一致

只有当两种办理类型的材料配置中使用了**相同的 materialId** 时，图片才会被保留。

**示例：**

✅ **正确配置（图片会保留）：**
```javascript
学生签证：
{ materialId: "passport", name: "护照首页" }

商务签证：
{ materialId: "passport", name: "护照首页" }
// materialId相同，图片会保留
```

❌ **错误配置（图片不会保留）：**
```javascript
学生签证：
{ materialId: "passport_student", name: "护照首页" }

商务签证：
{ materialId: "passport_business", name: "护照首页" }
// materialId不同，系统认为是不同材料，图片不会保留
```

### 2. 材料配置建议

为了最大化利用智能合并功能，在配置材料模板时：

✅ **推荐做法：**
- 通用材料使用统一的 materialId
- 例如：
  - `passport` - 护照首页（所有签证类型通用）
  - `photo` - 2寸白底照片（所有签证类型通用）
  - `id_card` - 身份证（中国签证通用）

❌ **不推荐做法：**
- 每种办理类型使用不同的 materialId
- 例如：
  - `passport_f1`, `passport_b1`, `passport_tourist` ...

### 3. 多人申请处理

- 材料按人员分别匹配
- 主申请人的材料不会影响同行人
- 每个人的材料独立保留或删除

### 4. 问题答案不保留

- ❌ 问题答案总是会被清空
- 因为不同办理类型的问题通常完全不同
- 无法像材料一样进行智能匹配

## 📝 常见问题

### Q1: 如何知道哪些材料的图片会被保留？

**答案：**
- 在模板配置中查看两种办理类型的材料清单
- 比较 `materialId` 字段
- 相同的 `materialId` = 图片会保留

### Q2: 保留的图片数量有限制吗？

**答案：**
- ❌ 没有限制
- 所有匹配材料的所有图片都会保留
- 包括多张图片的情况（allowMultiple 材料）

### Q3: 如果材料名称不同但ID相同会怎样？

**答案：**
- ✅ 图片会保留
- 材料名称会更新为新办理类型的名称
- 图片路径和数据不会改变

**示例：**
```javascript
原材料：
{ materialId: "passport", name: "护照首页", images: ["p1.jpg"] }

新材料模板：
{ materialId: "passport", name: "有效护照", ... }

结果：
{ materialId: "passport", name: "有效护照", images: ["p1.jpg"] }
// name更新，images保留
```

### Q4: 更换多次会丢失图片吗？

**答案：**
- 只要 `materialId` 一直匹配，图片就会一直保留
- 可以安全地多次更换办理类型
- 示例：学生签证 → 商务签证 → 旅游签证 → 学生签证（护照图片一直保留）

### Q5: 能否手动选择保留哪些材料？

**答案：**
- ❌ 当前版本不支持手动选择
- 系统自动根据 `materialId` 判断
- 如需保留特定材料，可以在更换前截图或下载

### Q6: 后台日志会显示保留了多少材料吗？

**答案：**
- ✅ 会显示
- 服务器日志：`已自动加载材料清单: 15 项，保留了 5 项已上传的材料`
- 处理记录：`办理类型从"学生签证"更换为"商务签证"，已智能合并材料（保留相同材料的图片），已清空问题答案`

## 🔍 调试和验证

### 如何验证材料是否正确保留？

1. **更换前记录**
   - 截图或记录已上传材料的数量
   - 记录每种材料的图片数量

2. **执行更换**
   - 点击"更换"按钮
   - 选择新的办理类型
   - 确认更换

3. **更换后检查**
   - 查看材料清单
   - 确认相同材料的图片仍然存在
   - 检查服务器日志中的保留数量

### 服务器日志示例

```bash
✅ 办理类型更换，已清空原问题答案，材料将智能合并
✅ 已自动加载问题模板: 8 个问题
✅ 已自动加载材料清单: 18 项，保留了 6 项已上传的材料
```

**解读：**
- 共生成 18 项材料（3个人 × 6种材料）
- 其中 6 项保留了原有的图片
- 12 项是新增的（未上传）
