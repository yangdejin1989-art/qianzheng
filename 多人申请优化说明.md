# 多人申请功能优化说明

## 📋 优化目标

实现签证多人申请时的智能填写功能，参考主流网站（12306、携程、Booking等）的做法：
- 区分共同问题和个人问题
- 同行人默认继承主申请人的答案
- 材料按人员分组显示
- 点击不同人员标签时，显示对应人员的材料和问答

### 4. 数据迁移脚本

为现有数据创建迁移脚本：

```javascript
// qianzheng/server/migrate_to_multi_person.js

const Application = require('./models/Application');

async function migrateApplications() {
  const applications = await Application.find({});
  
  for (const app of applications) {
    // 如果已经有新格式数据，跳过
    if (app.persons && app.persons.length > 0) continue;
    
    // 构建persons数组
    const persons = [
      { personId: 'main', name: app.name, role: 'main' }
    ];
    
    if (app.companions && app.companions.length > 0) {
      app.companions.forEach((name, i) => {
        persons.push({
          personId: `comp${i + 1}`,
          name,
          role: 'companion'
        });
      });
    }
    
    // 迁移材料数据
    const materialsByPerson = persons.map(person => ({
      personId: person.personId,
      materials: person.personId === 'main' ? app.materials || [] : []
    }));
    
    // 迁移问答数据
    const questionsByPerson = persons.map(person => ({
      personId: person.personId,
      questionsAnswers: person.personId === 'main' ? app.questionsAnswers || [] : []
    }));
    
    // 更新数据
    app.persons = persons;
    app.materialsByPerson = materialsByPerson;
    app.questionsByPerson = questionsByPerson;
    app.commonQuestions = [];
    
    await app.save();
    console.log(`✅ 迁移完成: ${app.applyCode}`);
  }
}

migrateApplications();
```

## 💡 使用示例

### 管理员查看多人申请

1. 打开申请详情页
2. 看到人员标签：`[张三(主申请人)] [李四] [王五]`
3. 点击"李四"
4. 材料区域显示：李四的护照、照片等
5. 问答区域显示：李四的个人信息
6. 共同信息区域显示：所有人的旅行目的、居住地址等

### 用户填写多人申请（简化版）

1. 填写主申请人信息
2. 添加同行人：李四、王五
3. 填写共同信息（自动应用到所有人）
4. 切换到李四标签
5. 点击"从主申请人复制"
6. 修改不同的信息（如护照号）
7. 切换到王五标签，重复操作
8. 提交
