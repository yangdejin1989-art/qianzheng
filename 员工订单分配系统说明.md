# 员工订单分配系统使用说明

## 功能概述

系统现在支持基于员工的订单分配功能，实现了以下核心功能：

1. **订单分配给员工** - 管理员可以将新订单分配给特定员工处理
2. **员工只能查看自己的订单** - 员工登录后只能看到分配给自己的订单
3. **管理员查看全部订单** - 管理员可以查看所有订单，包括已分配和未分配的
4. **订单负责人筛选** - 管理员可以按负责人筛选订单

## 系统变更说明

### 后端变更

#### 1. 数据库模型更新 (`Application.js`)
在订单模型中新增以下字段：
- `assignedTo`: 订单分配给的员工ID（引用Admin模型）
- `assignedBy`: 分配订单的管理员ID（引用Admin模型）
- `assignedAt`: 订单分配时间

#### 2. 新增认证中间件 (`authMiddleware.js`)
- `authenticate`: JWT认证中间件，验证token并获取当前登录用户
- `requireAdmin`: 要求管理员权限的中间件
- `requireAdminOrStaff`: 要求管理员或员工权限的中间件

#### 3. API接口更新

##### 修改的接口：
- **GET `/api/applications`** - 订单列表查询
  - 添加了认证和权限验证
  - 员工只能查看分配给自己的订单
  - 管理员可以查看所有订单
  - 支持 `assignedTo` 参数筛选（管理员专用）：
    - `all`: 查看所有订单
    - `unassigned`: 查看未分配的订单
    - `{员工ID}`: 查看分配给特定员工的订单

##### 新增的接口：
- **POST `/api/applications/:id/assign`** - 分配订单给员工（仅管理员）
  - 请求体：`{ assignedTo: "员工ID" }` （或 `null` 取消分配）
  - 返回：更新后的订单信息

- **GET `/api/admin/staff`** - 获取员工列表（仅管理员）
  - 返回：所有激活的管理员和员工账号列表

### 前端变更

#### 1. ApplicationManager 组件更新
- 新增员工列表状态和分配对话框
- 新增负责人筛选器（管理员可见）
- 表格新增"负责人"列，显示订单当前负责人
- 操作列新增"分配"按钮（管理员可见）
- 实现订单分配对话框，支持选择员工或取消分配

#### 2. AdminPanel 组件更新
- 传递 `currentUser` 参数给 ApplicationManager 组件
- 用于判断当前用户角色，控制界面元素显示

## 使用指南

### 管理员操作

#### 1. 查看所有订单
登录后台，在"申请管理"页面可以看到所有订单，包括：
- 每个订单的负责人显示在"负责人"列
- 未分配的订单显示"未分配"

#### 2. 分配订单给员工
- 在订单列表中，点击某个订单的"分配"按钮
- 在弹出的对话框中选择要分配的员工
- 点击"确认分配"完成分配
- 如需取消分配，选择"取消分配"选项

#### 3. 按负责人筛选订单
在筛选器中新增了"负责人"下拉菜单，可以选择：
- **全部负责人**: 查看所有订单
- **未分配**: 只查看未分配的订单
- **具体员工名**: 只查看分配给该员工的订单

### 员工操作

#### 1. 查看自己的订单
- 员工登录后，只能看到分配给自己的订单
- 无法看到其他员工或未分配的订单
- 不显示"负责人"筛选器和"分配"按钮

#### 2. 处理订单
员工可以正常查看和处理分配给自己的订单，包括：
- 查看订单详情
- 更新订单状态
- 添加反馈信息
- 处理材料审核等

## 注意事项

1. **身份验证要求**
   - 所有订单查询接口现在都需要有效的JWT token
   - 前端请求时必须在请求头中包含 `Authorization: Bearer {token}`

2. **新订单默认未分配**
   - 客户提交的新订单默认不分配给任何人
   - 管理员可以在后台手动分配给员工处理

3. **员工账号管理**
   - 需要先在"用户管理"中创建员工账号
   - 确保员工账号的 `isActive` 状态为启用

4. **分配记录**
   - 系统会记录谁（assignedBy）在什么时间（assignedAt）分配了订单
   - 便于追踪订单分配历史

## 技术细节

### JWT Token 格式
登录成功后返回的token包含以下信息：
```json
{
  "id": "用户ID",
  "username": "用户名",
  "role": "admin" 或 "staff"
}
```

### 订单查询过滤逻辑
- 员工：`filter.assignedTo = 当前员工ID`
- 管理员：根据 `assignedTo` 参数决定
  - 无参数或 `all`: 不添加过滤
  - `unassigned`: `filter.assignedTo = null`
  - 其他: `filter.assignedTo = 指定的员工ID`

## 常见问题

**Q: 现有的订单会分配给谁？**
A: 现有订单的 `assignedTo` 字段为 null（未分配），管理员可以手动分配。

**Q: 员工能否看到未分配的订单？**
A: 不能。员工只能看到明确分配给自己的订单。

**Q: 管理员自己能否被分配订单？**
A: 可以。管理员账号也可以作为订单负责人。

**Q: 能否将订单从一个员工转给另一个员工？**
A: 可以。点击"分配"按钮，选择新的负责人即可重新分配。

**Q: 取消分配后订单会怎样？**
A: 订单会回到"未分配"状态，员工无法再看到该订单，只有管理员能看到。
