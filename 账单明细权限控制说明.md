# 账单明细权限控制说明

## 功能概述

对订单详情中的"账单明细"部分进行了权限控制，实现了基于角色的功能访问限制：

- **管理员**：可以查看和编辑付款明细、成本费用和利润信息
- **员工**：只能查看和填写付款明细（入金），无法查看成本费用和利润

## 修改内容

### 1. BillingManager 组件（`qianzheng/client/src/components/BillingManager.js`）

#### 新增参数
```javascript
const BillingManager = ({ applicationId, currentUser }) => {
  // 判断是否为管理员
  const isAdmin = currentUser?.role === 'admin';
  // ...
}
```

#### 成本费用部分权限控制
```javascript
{/* 成本费用 - 仅管理员可见 */}
{isAdmin && (
  <div className="billing-section">
    {/* 成本费用内容 */}
  </div>
)}
```

### 2. ApplicationDetail 组件（`qianzheng/client/src/ApplicationDetail.js`）

#### 获取当前用户信息
```javascript
function ApplicationDetail({ id, onBack }) {
  // 获取当前登录用户信息
  const currentUser = JSON.parse(localStorage.getItem('userInfo') || '{}');
  // ...
}
```

#### 传递用户信息给 BillingManager
```javascript
<BillingManager applicationId={id} currentUser={currentUser} />
```

## 权限说明

### 管理员（role: 'admin'）

**可见内容**：
- ✅ 付款明细
  - 查看所有支付记录
  - 添加新支付记录
  - 删除支付记录
  
- ✅ 成本费用
  - 查看成本金额和备注
  - 编辑成本费用
  
- ✅ 利润统计
  - 查看利润金额
  - 查看利润率

**操作权限**：
- ✅ 添加/删除付款记录
- ✅ 编辑成本费用
- ✅ 查看完整的财务统计

### 员工（role: 'staff'）

**可见内容**：
- ✅ 付款明细
  - 查看所有支付记录
  - 添加新支付记录
  - 删除支付记录

- ❌ 成本费用（完全隐藏）
- ❌ 利润统计（完全隐藏）

**操作权限**：
- ✅ 添加/删除付款记录
- ❌ 无法查看或编辑成本费用
- ❌ 无法查看利润信息

## 实现原理

### 权限判断逻辑

1. **用户信息获取**
   - 从 `localStorage` 中获取 `userInfo`
   - 包含用户的 `role`、`username` 等信息

2. **权限检查**
   ```javascript
   const isAdmin = currentUser?.role === 'admin';
   ```

3. **条件渲染**
   - 使用 React 的条件渲染 `{isAdmin && <Component />}`
   - 如果不是管理员，成本费用部分完全不渲染（DOM 中不存在）

### 安全性

#### 前端安全
- ✅ 成本费用部分在员工账户下不渲染到 DOM
- ✅ 无法通过浏览器开发者工具查看隐藏内容
- ✅ 组件级别的权限控制

#### 后端安全
- ⚠️ 前端权限控制仅用于界面展示
- ⚠️ 后端 API 应该有相应的权限验证
- 💡 建议：后端 `/api/applications/:id/billing/cost` 接口也应添加管理员权限检查

## 使用说明

### 管理员操作

1. **查看账单明细**
   - 登录管理员账号
   - 进入订单详情页面
   - 可以看到完整的账单信息，包括付款、成本和利润

2. **记录付款**
   - 点击"添加支付"按钮
   - 填写金额、日期、支付人和备注
   - 确认添加

3. **编辑成本**
   - 点击"编辑成本"按钮
   - 填写成本金额和备注
   - 保存修改
   - 利润会自动计算并更新

### 员工操作

1. **查看付款明细**
   - 登录员工账号
   - 进入订单详情页面
   - 只能看到付款明细部分

2. **记录付款**
   - 点击"添加支付"按钮
   - 填写金额、日期、支付人和备注
   - 确认添加

3. **限制**
   - 无法查看成本费用
   - 无法查看利润统计
   - 无法看到成本相关的任何信息

## 注意事项

### 开发者注意

1. **currentUser 传递**
   - 必须确保在使用 BillingManager 的地方传递 currentUser
   - 如果未传递，默认当作非管理员处理

2. **localStorage 依赖**
   - 依赖 `localStorage.getItem('userInfo')`
   - 确保登录时正确存储用户信息

3. **后端 API 权限**
   - 前端权限仅用于 UI 控制
   - 后端应该有独立的权限验证
   - 防止通过 API 直接访问敏感信息

### 用户注意

1. **员工账号限制**
   - 员工登录后无法看到成本和利润
   - 这是正常的系统设计，不是 bug

2. **管理员权限**
   - 只有管理员账号可以管理成本
   - 建议妥善保管管理员账号
