# 多人申请功能 - 快速开始指南

## ✅ 当前可用功能

## 📋 数据结构说明

### 新增字段

```javascript
{
  // 人员列表
  persons: [
    { personId: 'main', name: '张三', role: 'main' },
    { personId: 'comp1', name: '李四', role: 'companion' }
  ],
  
  // 共同问题
  commonQuestions: [
    { questionId: 'q1', questionText: '旅行目的', answer: '商务' }
  ],
  
  // 按人员分组的问答
  questionsByPerson: [
    {
      personId: 'main',
      questionsAnswers: [
        { 
          questionId: 'q2', 
          questionText: '出生日期', 
          answer: '1990-01-01',
          inheritedFrom: null  // null表示非继承
        }
      ]
    },
    {
      personId: 'comp1',
      questionsAnswers: [
        { 
          questionId: 'q2', 
          questionText: '出生日期', 
          answer: '1990-01-01',
          inheritedFrom: 'main'  // 继承自主申请人
        }
      ]
    }
  ],
  
  // 按人员分组的材料
  materialsByPerson: [
    {
      personId: 'main',
      materials: [
        { materialId: 'm1', materialName: '护照', images: [...] }
      ]
    },
    {
      personId: 'comp1',
      materials: []
    }
  ]
}
```

## ⚠️ 注意事项

### 1. 兼容性
- ✅ 保留了旧字段 `materials` 和 `questionsAnswers`
- ✅ 未迁移的旧数据仍可正常使用
- ✅ 新旧数据可以混合存在

### 2. 迁移时机
- 建议在**低峰期**运行迁移脚本
- 迁移过程约需 1-5 分钟（取决于数据量）
- 迁移过程中数据库仍可正常使用

### 3. 数据安全
- 迁移脚本**不会删除**旧数据
- 仅添加新字段，不修改原有字段
- 可随时回滚到旧版本

## 🎯 应用场景

### 场景1：家庭签证申请
```
一家三口申请旅游签证：
- 共同信息：旅行目的、酒店预订、行程单
- 个人信息：护照、照片、出生日期
→ 共同信息填一次，个人信息各自填写
```

### 场景2：公司团队签证
```
公司5人商务签证：
- 共同信息：公司地址、邀请函、行程
- 个人信息：护照、工作证明、职位
→ HR填写共同信息，员工各自补充个人材料
```

### 场景3：朋友结伴申请
```
朋友3人旅游签证：
- 部分共同：目的地、酒店（2人同住）
- 部分独立：另1人单独酒店
→ 灵活选择哪些问题应用到哪些人
```

### 常见问题
1. **迁移后旧数据看不到？**
   - 检查是否运行了迁移脚本
   - 查看浏览器控制台是否有错误

2. **点击标签没反应？**
   - 刷新浏览器缓存
   - 检查 `materialsByPerson` 字段是否存在

3. **继承标识不显示？**
   - 检查 `inheritedFrom` 字段
   - 目前旧数据迁移后默认无继承标识
