# 邮件功能问题解决指南

## 🚨 当前问题

你遇到的错误：
- ❌ 邮件发送失败: 网络错误
- 获取邮件配置失败: SyntaxError: Unexpected token '<'
- API 接口 404 错误

## 🔍 问题原因

1. **环境变量未设置**：后端服务启动时没有正确的环境变量
2. **API 接口未加载**：新的邮件 API 接口没有正确加载
3. **服务需要重启**：修改后的代码需要重启服务才能生效

## 🛠️ 解决方案

### 方案一：使用重启脚本（推荐）

1. **关闭当前的后端服务**
2. **运行 `重启邮件功能.bat`**
3. **等待服务启动完成**
4. **测试邮件功能**

### 方案二：手动重启

1. **关闭当前的后端服务**
2. **在 `server` 目录下运行：**
   ```bash
   set EMAIL_USER=jishu2020_service@163.com
   set EMAIL_PASS=QDyQgPu328neKbEE
   set ENABLE_STATUS_EMAILS=true
   npm start
   ```

### 方案三：使用原始启动脚本

1. **关闭当前服务**
2. **运行 `163邮箱一键启动.bat`**

## ✅ 验证步骤

重启后，按以下步骤验证：

1. **检查后端控制台输出**
   - 应该看到环境变量设置信息
   - 应该看到 "邮件传输器连接成功"

2. **测试 API 接口**
   - 访问：http://localhost:5000/api/email-config
   - 应该返回 JSON 数据，不是 HTML

3. **测试邮件功能**
   - 在管理后台查看订单详情
   - 使用邮件管理组件发送邮件
   - 修改申请状态测试自动发送

## 🔧 常见问题

### Q: 为什么还是显示"网络错误"？
A: 确保后端服务已经重启，并且环境变量正确设置

### Q: API 接口返回 HTML 而不是 JSON？
A: 说明后端服务没有加载新的代码，需要重启

### Q: 环境变量设置在哪里？
A: 在启动脚本中，或者在启动后端服务前手动设置
