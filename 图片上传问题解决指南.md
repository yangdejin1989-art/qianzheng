# 图片上传问题解决指南

## 问题描述

在使用WangEditor时，图片上传功能出现错误：
```
ERROR
Upload error
    at Object.getResponseError
```

## 问题原因

1. **后端缺少上传接口**：原项目没有 `/api/upload` 接口
2. **配置过于复杂**：WangEditor的图片上传配置包含了一些不必要的参数
3. **跨域问题**：可能存在CORS配置问题

## 解决方案

### 1. 添加后端上传接口 ✅

已在 `server/index.js` 中添加了通用图片上传接口：

```javascript
// 通用图片上传接口
app.post('/api/upload', upload.single('image'), async (req, res) => {
  try {
    if (!req.file) {
      return res.status(400).json({ success: false, message: '没有上传文件' });
    }
    
    const imageUrl = '/uploads/' + req.file.filename;
    res.json({
      success: true,
      url: imageUrl,
      alt: req.file.originalname,
      message: '图片上传成功'
    });
  } catch (error) {
    console.error('图片上传错误:', error);
    res.status(500).json({ success: false, message: '图片上传失败' });
  }
});
```

### 2. 简化WangEditor配置 ✅

更新了 `WangEditor.js` 的配置，移除了不必要的参数：

```javascript
const editorConfig = {
  placeholder: placeholder,
  MENU_CONF: {
    uploadImage: {
      server: 'http://localhost:5000/api/upload',
      fieldName: 'image',
      maxFileSize: 5 * 1024 * 1024, // 5MB
      allowedFileTypes: ['image/*'],
      onBeforeUpload(file) {
        console.log('准备上传文件:', file.name, file.size);
        return file;
      },
      onSuccess(file, res) {
        console.log('上传成功:', res);
      },
      onFailed(file, res) {
        console.error('上传失败:', res);
      },
      onError(file, err, res) {
        console.error('上传错误:', err);
      },
      customInsert(res, insertFn) {
        console.log('处理上传结果:', res);
        try {
          if (res && res.success && res.url) {
            const imageUrl = res.url.startsWith('http') ? res.url : `http://localhost:5000${res.url}`;
            insertFn(imageUrl, res.alt || '', res.href || '');
          } else {
            console.error('图片上传返回格式错误:', res);
          }
        } catch (error) {
          console.error('插入图片时出错:', error);
        }
      }
    }
  }
};
```

### 3. 提供多个版本 ✅

创建了三个版本的WangEditor：

1. **WangEditor.js** - 完整版本，包含图片上传功能
2. **WangEditorSimple.js** - 简化版本，更简单的配置
3. **WangEditorNoUpload.js** - 无上传版本，不包含图片上传功能

### 4. 测试页面 ✅

创建了 `TestUpload.js` 测试页面，可以对比不同版本的功能。

## 使用建议

### 如果需要图片上传功能

使用 `WangEditor.js`：

```jsx
import WangEditor from './WangEditor';

<WangEditor
  value={content}
  onChange={setContent}
  placeholder="请输入内容..."
  height={300}
/>
```

### 如果不需要图片上传功能

使用 `WangEditorNoUpload.js`：

```jsx
import WangEditorNoUpload from './WangEditorNoUpload';

<WangEditorNoUpload
  value={content}
  onChange={setContent}
  placeholder="请输入内容..."
  height={300}
/>
```

## 故障排除

### 1. 检查服务器是否启动

确保后端服务器正在运行：
```bash
cd server
node index.js
```

### 2. 检查网络请求

打开浏览器开发者工具，查看Network标签页，检查上传请求是否正常。

### 3. 检查控制台错误

查看浏览器控制台是否有错误信息。

### 4. 测试上传接口

可以直接测试上传接口：
```bash
curl -X POST -F "image=@test.jpg" http://localhost:5000/api/upload
```

## 配置说明

### 上传接口配置

- **服务器地址**: `http://localhost:5000/api/upload`
- **字段名**: `image`
- **文件大小限制**: 5MB
- **文件类型**: 图片文件 (`image/*`)

### 返回格式

服务器返回的JSON格式：
```json
{
  "success": true,
  "url": "/uploads/filename.jpg",
  "alt": "original_filename.jpg",
  "message": "图片上传成功"
}
```

## 注意事项

1. 确保 `uploads` 目录存在且有写入权限
2. 图片文件会保存在 `server/uploads/` 目录下
3. 上传的图片可以通过 `http://localhost:5000/uploads/filename.jpg` 访问
4. 建议在生产环境中配置适当的文件大小限制和安全检查
